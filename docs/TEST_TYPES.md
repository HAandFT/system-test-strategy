# 🧪 TEST TYPES

이 문서는 개발 및 운영 환경에서 사용되는 다양한 테스트의 종류를 설명합니다.  
각 테스트는 목적, 범위, 예시 상황에 따라 구분되며, 테스트 전략 수립의 기준이 됩니다.

---

## ✅ 테스트 종류 요약

| 분류 기준 | 테스트 종류 | 설명 |
|-----------|-------------|------|
| 🔍 단위(Unit) | 단위 테스트 (Unit Test) | 함수나 메서드 하나의 동작을 검증. 빠르고 작음. 예: `계산기.add(1,2)`가 3을 리턴하는지 |
| 🧱 통합(Integration) | 통합 테스트 (Integration Test) | 여러 컴포넌트나 모듈이 잘 작동하는지 검증. 예: DB, API, 메시지 큐 연동 테스트 |
| 🏗 E2E(End-to-End) | 종단 테스트 (E2E Test) | UI부터 DB까지 전체 흐름 검증. 예: 회원가입 → 로그인 → 글쓰기 |
| 🎨 UI | UI 테스트 | 버튼 클릭, 텍스트 렌더링 확인. 예: "비밀번호가 틀렸습니다" 메시지 표시 여부 |
| 🌐 API | API 테스트 | REST API 응답 형식, 상태코드, 데이터 확인. 예: `GET /users` → 200 OK & 유저 목록 |
| 🔁 회귀(Regression) | 회귀 테스트 | 기존 기능이 새로운 코드로 인해 깨지지 않았는지 확인 |
| 📦 Smoke | Smoke Test | 배포 후 가장 기본적인 기능만 빠르게 확인. 예: 서버 실행 여부, API 응답 200 |
| 🔒 보안(Security) | 보안 테스트 | 인증/인가, XSS, CSRF, 포트, 보안그룹 등 방어 점검 |
| 💣 부하/성능 | Load/Stress Test | 많은 사용자가 동시에 요청할 때 정상 작동 여부 검증 |
| 🔄 복구/고가용성 | HA Test / Chaos Test | 일부 시스템이 죽었을 때 장애 복구 여부 확인 |
| 🔍 정적 분석 | 정적 코드 분석 | 실행 없이 코드 구조나 문법을 분석. 예: SonarQube, ESLint |
| 👁 시각적 회귀 | Visual Regression Test | UI 변화가 없는지 이미지 기반 비교. 예: 버튼 위치, 색상 변화 확인 |

---

## 📌 참고 사항

- 테스트 종류는 겹칠 수 있으며, 목적에 따라 복합적으로 구성될 수 있음
- 실제 프로젝트에서는 여러 테스트를 **자동화**하여 통합 운영하는 것이 바람직
- 보안, 복구, Chaos 테스트는 실서비스 안정성과 직접 연결되는 고급 테스트 항목
